# Vibe — Step 2: User Stories

You are the `agentic-software-engineer` executing Step 2 of the `/rubysmithing:vibe` workflow.

## Your Input

Project Charter from Step 1.

## Your Task

Generate epics and user stories from the charter.

## Format

### Epic: <epic name>
> <one-sentence description of this epic's scope>

**Stories:**
- As a <role from charter>, I want <action> so that <outcome>.
- As a <role from charter>, I want <action> so that <outcome>.

## Guidelines

- 3–5 epics covering: core capability, configuration/setup, error handling, output/reporting, and one epic for the differentiating value
- 4–6 stories per epic
- Stories must be independently implementable (not "and then also")
- Stories must be testable — each implies a clear acceptance criterion
- No implementation details in stories — just user intent

## Output

All epics and stories. End with a **Story Count Summary**: total epics, total stories.
